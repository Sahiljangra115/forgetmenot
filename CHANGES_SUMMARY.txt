ISSUE 1: Frontend UI Inconsistency Bug
--------------------------------------
PROBLEM:
  - localhost:3000 showed updated UI and worked fine
  - 192.168.229.124:3000 showed older/different UI
  - Network access couldn't connect to backend
  - Features and UI were completely different

ROOT CAUSE:
  - getBackendUrl() hardcoded: "port === '3000' ? http://localhost:8000 : ''"
  - When accessing via network IP, returned empty string
  - All API calls failed, frontend fell back to cached content

SOLUTION:
  - Use window.location.hostname (dynamic) instead of hardcoded "localhost"
  - Construct URL from current host: http://[user's-host]:8000
  - Both localhost and network access now work identically

  ✓ frontend/app/camera/page.tsx

    - Fixed getWsUrl() function
  
  ✓ frontend/app/login/page.tsx

    - Changed CORS allow_origins from hardcoded list to "*"


ISSUE 2: No User-Provided LLM API Key Support
----------------------------------------------
PROBLEM:
  - Hackathon product couldn't be deployed by users
  - Required sharing private API keys in .env files
  - No way for users to use their own LLM accounts

SOLUTION:
  - Added "LLM API Settings" UI panel on camera page
  - Users can select provider and enter API key
  - Backend now supports 6 major LLM providers
  - API keys stored locally only, never persisted on server

SUPPORTED PROVIDERS:
  1. OpenAI (GPT-4o, GPT-4 Turbo, etc)
  2. Anthropic (Claude 3.5 Sonnet, etc)
  3. OpenRouter (Multi-provider gateway)
  4. DeepSeek (DeepSeek Chat)
  5. Google (Gemini 2.0 Flash)
  6. Ollama (Local LLM - no API key needed)
